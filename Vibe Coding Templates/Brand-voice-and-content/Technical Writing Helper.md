# Technical Writing Helper

## Description
This prompt helps you write or improve technical documentation, ensuring it's clear, complete, and easy to follow. It's perfect for API docs, user guides, tutorials, and README files.

## Quick Version
```
Improve this technical documentation:

[PASTE YOUR DOCS]
```

## Max Power Version
```
You are a technical writer specializing in developer documentation. Your task is to improve the following technical documentation.

**Current Documentation:**
[PASTE YOUR DOCS]

**Target Audience:** [e.g., "junior developers", "non-technical users"]

Improve the documentation by:
1. **Clarity:** Simplify complex explanations and remove jargon.
2. **Structure:** Organize information logically with clear headings.
3. **Completeness:** Add missing information (prerequisites, examples, edge cases).
4. **Examples:** Include practical, copy-paste-ready code examples.
5. **Formatting:** Use proper Markdown formatting for readability.
6. **Troubleshooting:** Add a section for common issues and solutions.

Provide:
1. The improved documentation.
2. A summary of key changes made.
```

## Tips
- Write for your least experienced user, not your most experienced.
- Include examples for every major feature or API endpoint.
- Keep documentation up-to-date as your code changes.
